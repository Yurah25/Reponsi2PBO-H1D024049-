Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

Apa yang dimaksud dengan constructor overloading dalam pemrograman berorientasi objek?
Bagaimana cara Java membedakan constructor yang memiliki nama sama namun parameter berbeda?
Apa perbedaan antara mengakses variabel secara langsung (misalnya: objek.variabel = nilai) dengan menggunakan method setter?
Jelaskan konsep encapsulation dan bagaimana method dapat membantu melindungi data dalam sebuah class!
Mengapa constructor tidak memiliki return type, bahkan tidak menggunakan kata kunci void?

Jawaban
1. constructur overloading adalah membuat constructur yang banyak sesuai kebutuhan atau kondisi dengan class yang sama memakai parameter berbeda.
2. denagn parameter urutannya, tipe data dan jumlahnya.
3. perbedaan mengakses langsung dan setter yaitu keamanan, setter bisa menambahkan sebuah validasi logika pengecekan dan cdigunakan untuk private tipe data agar aman tidak bisa diakses package lain secara langsung.
4. konsep membungkus data attribut dan method  menjadi satu di sebuah class dengan menyembunikan tincian implementasi dengan private yang menggunakan method setter getter
5. karena constructor bukan untuk dipanggil melakukan aksi void dan tidak melakukan perhitungan dengan return , jadi constructur hanya melakaukan inisialisasi untuk objek yang baru dibuat.
